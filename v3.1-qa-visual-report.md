## Benchmark comparison: apushscorecalculator.us

Benchmark strengths observed:

- Calculator and result visualization are tightly coupled on one APUSH-focused page.
- Section contribution breakdown is explicit and updates around the score result.
- APUSH-specific content is deep: digital Bluebook facts, section format, college credit, distribution, study resources, FAQs.

Our v3.1 strengths vs benchmark:

- Visual system is more modern: warmer background, stronger cards, higher perceived polish, better CTA hierarchy.
- Homepage is broader and better structured for multi-subject AP calculators.
- APUSH layout has stronger editorial sectioning and richer dashboard modules than prior versions.
- Mobile input boxes are comfortable and no overflow was found at 360/390/414.

Our v3.1 gaps vs benchmark:

- Benchmark's result visuals are more clearly tied to calculator state. Our text result updates, but the dashboard/target visualizations remain static sample guidance.
- Benchmark has more APUSH-specific depth and historical-resource flavor; our page still feels like a polished general calculator with APUSH text rather than a deeply historical/APUSH-branded experience.
- Homepage hub preview category links visually run together on desktop (`AP ChemAP Calc ABAP Bio` appearance) and need chip spacing/separators.

## Issues found

### Issue 1 — APUSH score dashboard / target cards stay stale after calculator input

Severity: High
Category: Functional / UX / Visual data consistency
URL: http://127.0.0.1:4191/apush-score-calculator/
Screenshot: /root/.hermes/kanban/boards/outbound-sites/workspaces/t_830a4536/v3.1-qa-screenshots/apush-360.png and /root/.hermes/profiles/qa/cache/screenshots/browser_screenshot_3fba04fa1dc3487187cbc399e204bf33.png

Steps to reproduce:

1. Open `/apush-score-calculator/`.
2. Enter MCQ `42`, SAQ `7`, DBQ `5`, LEQ `4`.
3. Observe the main result card and the score-band dashboard / target cards.

Expected:

- Result card, score-band marker, and target cards should all reflect the same computed result.
- If the result is AP 5 with `Estimated Composite Score: 96 / 130`, the target cards should not say Target 5 is still 2 composite points away.

Actual:

- Main result updates to `Estimated AP score 5` and `Estimated Composite Score: 96 / 130`.
- Score-band dashboard/target cards still show sample guidance: Target 5 `2 composite pts away`, `Near cutoff`.
- Hero preview also remains `Estimated AP 4`, `90/130 composite`, which may be acceptable as a static preview only if clearly labeled; the main dashboard below the calculator reads like user-specific output and is misleading.

Recommendation:

- Bind the score-band marker and target cards to the same calculator state as the result card.
- Or explicitly label all non-dynamic dashboards as `Example preview` and move dynamic target guidance into/near the result card only.

### Issue 2 — Homepage hub preview category links lack visible separation on desktop

Severity: Medium
Category: Visual / UX
URL: http://127.0.0.1:4191/
Screenshot: /root/.hermes/kanban/boards/outbound-sites/workspaces/t_830a4536/v3.1-qa-screenshots/home-desktop.png

2. Scroll to `Browse calculators by category` / Hub Preview.
3. Inspect category card links.

Expected:

- Links should read as separate chips or separated inline links.

Actual:

- Links visually concatenate, e.g. `AP ChemAP Calc ABAP Bio`, `AP LangAP Lit`, `APUSHAP GovAP Psych`.

Recommendation:

- Render as chips/pills with margins, or add separators and spacing.

### Issue 3 — Mobile homepage hero is attached as requested but feels overloaded at 360px

Severity: Low
Category: Visual / UX
URL: http://127.0.0.1:4191/
Screenshot: /root/.hermes/kanban/boards/outbound-sites/workspaces/t_830a4536/v3.1-qa-screenshots/home-360.png

Observation:

- The `Popular starting points` block is attached to the hero at 360/390/414, satisfying the acceptance criterion.
- However, the full hero becomes very tall: headline, copy, CTAs, disclaimer, dashboard card, and three popular cards all sit inside one large flow before the subject matrix.
- The uppercase `POPULAR STARTING POINTS` eyebrow wraps awkwardly at 360 (`POPULAR STARTING` / `POINTS`).

Recommendation:

- Keep the visual connection, but reduce mobile hero density: shorten copy, reduce dashboard preview height, or use a compact horizontal/stacked popular-card style.
- Consider changing the eyebrow to title case on mobile or reducing letter spacing.

### Issue 4 — Product visual identity is polished but still light on APUSH-specific assets

Severity: Low
Category: Visual / Brand
URL: http://127.0.0.1:4191/apush-score-calculator/
Screenshot: /root/.hermes/kanban/boards/outbound-sites/workspaces/t_830a4536/v3.1-qa-screenshots/apush-desktop.png

Observation:

- The page feels like a credible education SaaS calculator, with good cards, bands, and charts.
- Compared with the APUSH benchmark content depth, it could use more APUSH/history-specific visual flavor.

Recommendation:

- Add subtle APUSH/history motifs: timeline line, map contour, document/parchment accents, section icons for MCQ/SAQ/DBQ/LEQ, or historical note cards.
- Do not overdecorate; keep calculator clarity first.
