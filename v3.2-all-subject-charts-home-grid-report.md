# APSC v3.2 All-Subject Charts + Homepage Grid Report

Date: 2026-05-11
Workspace: /root/.hermes/kanban/boards/outbound-sites/workspaces/t_830a4536
Production deploy: Not performed

## Task
User feedback from preview 3000 reported that only the APUSH page had chart/dashboard visuals, while the other subject calculator pages felt inconsistent. The homepage subject card grid also laid out as 4/4/1 on desktop.

## Changes implemented

### 1. Shared visual score dashboard for all subject pages
- Updated `src/site.js` so every subject calculator page now renders the shared dashboard in the hero:
  - `score-dashboard-visual`
  - `data-score-dashboard`
  - `data-dashboard-score`
  - `data-dashboard-composite`
  - `data-score-band-strip`
  - `data-target-strip`
- The dashboard uses each subject's own `cutoffs` and `maxComposite` / `displayMaxComposite` from `src/scoreEngine.js`.
- APUSH keeps backward-compatible `data-apush-dashboard` markers for existing sync/tests, but the new shared marker is `data-score-dashboard`.

### 2. Score band dashboard section on every subject page
- Moved the score band strip + target 3/4/5 card visualization out of APUSH-only rendering.
- Every subject now has a dedicated `Score band dashboard` section after the calculator workspace.
- The copy is subject-specific, e.g. `See the AP Lang score bands...`, `See the AP Chem score bands...`.
- The band legend reflects subject-specific ranges such as each AP 1–5 cutoff.

### 3. APUSH-specific modules preserved
- Kept APUSH-only section-weighting module.
- Kept APUSH-only history/practice modules:
  - APUSH exam structure
  - What to practice next
  - section weighting chart
- Existing APUSH dashboard synchronization still passes via legacy APUSH attributes plus shared dashboard attributes.

### 4. Homepage subject matrix redesigned
- Replaced the previous single `auto-fit` 9-card subject matrix that could render as 4/4/1 on desktop.
- New structure:
  - one featured APUSH row/card (`featured-subject-row`)
  - one balanced 8-card grid (`subject-grid-balanced`) for all other subjects
- Desktop verification shows balanced four-column grid for the 8 cards: 4x2.
- Responsive CSS collapses to 2 columns at tablet width and 1 column on mobile.

### 5. Runtime sync updated
- Updated `scripts/build.js` app bundle so calculator input changes synchronize any `[data-score-dashboard]` dashboard.
- APUSH legacy `[data-apush-dashboard]` selectors are also still supported.

### 6. Tests updated
Added v3.2 regression tests in `test/site.test.js`:
- every subject page includes shared dashboard/score-band/target-strip markers
- every subject dashboard uses subject-specific score-band ranges
- homepage has one featured APUSH card plus a balanced 8-card grid
