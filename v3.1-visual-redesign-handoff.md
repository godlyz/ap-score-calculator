# AP Score Calculator v3.1 — Visual-Rich Redesign Handoff

Status: Ready for development
Date: 2026-05-11
Owner: Content / Design handoff
Primary reference: https://apushscorecalculator.us/

---

## 1. Redesign decision

Do not patch the existing APUSH page.
Do not keep the current visual skeleton and try to fix spacing only.
This v3.1 is a visible redesign, not a CSS cleanup.

The user rejected v3 because it still felt too close to the first version and did not have enough competitive visual substance. The new direction must explicitly solve:

- APUSH width and vertical rhythm inconsistency
- “Enter scores” label wrapping
- Homepage “Popular starting points” block detaching from hero when the viewport narrows
- Mobile score inputs being too small to read and tap comfortably
- The page still looking too similar to the first version
- Lack of charts, color-coded ranges, and visual assets

The APUSH page should now feel like a competitive subject-specific education product with stronger visual storytelling, not just a calculator page with explanatory copy.

---

## 2. Design goal

Make the APUSH experience feel:

- more editorial
- more data-rich
- more colorful without becoming playful
- more confident and competitive
- more like a complete subject product page
- more distinct from the previous version

The key product story is:

1. browse the site from the homepage
2. jump into APUSH from a clearly attached featured path
3. use a visually strong calculator workspace
4. read a score result panel that feels like an insight dashboard
5. continue into charts, score bands, and study guidance

---

## 3. What to borrow from apushscorecalculator.us

Borrow these behaviors:

- Single-subject clarity
- Calculator-first page rhythm
- Transparent scoring explanation
- Study guidance near the calculator
- FAQ and SEO content that answers real questions
- A page that feels purpose-built for APUSH search intent

Borrow the structure, not the styling.

---

## 4. What to improve beyond the competitor

We should exceed the competitor in these areas:

- stronger visual hierarchy
- richer chart and score-band presentation
- more color-coded interpretation
- stronger homepage to hub to APUSH continuity
- stronger mobile readability
- more polished result panel design
- more varied section backgrounds
- more useful visual assets instead of plain text blocks
- better rhythm between calculator, charts, guide, and FAQ

---

## 5. Design direction

Recommended direction: Editorial dashboard for students

This is not a generic SaaS look and not a plain white education template.
It should mix:

- editorial spacing and typography
- dashboard-style score visualization
- academic trust cues
- warm but restrained color accents

Visual tone:

- calm
- credible
- structured
- slightly premium
- visually rich but not flashy

---

Use a richer system than the previous version.
The old palette was too quiet and not competitive enough.

### Core palette

- Background: `#F6F3EC`
- Surface: `#FFFFFF`
- Surface Alt: `#EEF2F7`
- Ink: `#142033`
- Muted text: `#5F6B7A`
- Border: `#D8E0EA`

### Brand / accent palette

- Primary Navy: `#1D3557`
- Secondary Teal: `#2A9D8F`
- Accent Amber: `#F4A261`
- Alert Coral: `#E76F51`
- Success Green: `#4DAA57`
- Highlight Gold: `#E9C46A`

### Score band colors

Use explicit color coding for AP ranges:

- 5 band: `#2A9D8F`
- 4 band: `#4DAA57`
- 3 band: `#E9C46A`
- 2 band: `#F4A261`
- 1 band: `#E76F51`

Important:
- do not use a flat blue-only system
- do not use purple-blue gradient SaaS styling
- do not make all cards the same neutral gray
- score colors must show up in the chart, legend, badges, and result panel

---

## 7. Typography

Do not use Inter / Roboto / Arial as the primary look.

Recommended pairing:

- Display: `Sora`
- Body: `DM Sans`
- Optional numeric / data accent: `JetBrains Mono` for small score labels only

Typography rules:

- H1 should fit in one or two lines on desktop
- H2/H3 must keep strong vertical rhythm
- Numeric score elements should feel bold and legible
- All input labels must remain unwrapped at common widths unless intentionally stacked on mobile
- Mobile body text should never drop below 16px
- Mobile input text should be 16px or larger

## 8. Homepage → Hub → APUSH continuity

This is a core redesign requirement.
The featured APUSH path must look intentionally attached to the hero, not like a detached module.

### Homepage layout rule

Hero and “Popular starting points” must be treated as one visual system.

Do not let the featured block float away when the page narrows.

Desktop behavior:

- hero copy on left
- visual / preview / score card on right
- Popular starting points anchored directly beneath hero content or integrated into the hero lower band
- featured block should share the same container width and alignment as the hero

Tablet and narrow desktop behavior:

- collapse the hero and popular block into one stacked group
- keep shared background and border treatment
- do not separate them into disconnected sections

Mobile behavior:

- hero
- featured APUSH path
- popular starting points
should read as one continuous block

The user should not feel that the home page is a marketing block followed by a random list.

## 10. Hero treatment

The hero must do more than introduce the page.
It should establish the visual system and score language immediately.

### Hero content

- H1: APUSH Score Calculator 2026
- Supporting line: estimate your AP U.S. History score from MCQ, SAQ, DBQ, and LEQ
- Primary CTA: Estimate my APUSH score
- Secondary CTA: See scoring bands
- Trust line: free, browser-local, unofficial, not affiliated with College Board

### Hero visual treatment

Use a split hero on desktop:

- left: copy + CTA + trust row
- right: a live-looking score visualization card or a stacked preview of score bands

The right-side visual should not be decorative only.
It should look like a preview of the calculator result state.

Possible visual motifs:

- score band arc
- stacked result cards
- bar chart of section weights
- score range mini dashboard

Avoid a generic abstract blob.

---

## 11. Calculator workspace layout

This is the core of the page and must feel stronger than the previous version.

### Desktop layout

Use a 2-column workspace:

- left column: inputs
- right column: sticky or visually persistent result panel

Suggested proportions:

- left: 58%
- right: 42%

The width of the calculator area must stay stable across the common desktop breakpoints.
The APUSH page should not feel narrower than the surrounding sections.

### Vertical rhythm rule

Use consistent spacing tokens inside the calculator:

- section spacing: 24–32px
- card spacing: 16–20px
- field spacing: 12–16px
- group spacing: 20–24px

Do not allow random collapses between cards.
Every major block should feel aligned to the same rhythm.

### Input card treatment

Each input row must have:

- clear label
- helper line with range
- score input control
- optional mini legend or hint

Input labels should be short.
Do not let labels wrap unnecessarily.

#### Fix for “Enter scores” wrapping

If a label or row header such as “Enter scores” is wrapping awkwardly:

- give it a stronger min-width on desktop
- keep it on a single line unless the screen is genuinely narrow
- if needed, move the helper text below rather than wrapping the headline label
- never compress the whole card just to keep the label in one line

### Input labels

The APUSH input labels should be presented like this:

- MCQ correct
- SAQ total points
- DBQ score
- LEQ score

Avoid long helper words in the label itself.
Use helper text below.

### Desktop input field sizing

- field height: 48px minimum
- tap target height: 44px minimum
- number text size: 18px preferred on desktop
- label size: 14–15px
- helper size: 12–13px

## 12. Result panel design

The result panel should feel like an insight dashboard, not a plain summary box.

### Required result content

- estimated AP score
- estimated composite score
- gap to 3 / 4 / 5
- confidence label
- near-cutoff warning state when relevant
- short explanation of what drives the estimate

### Result panel visual structure

Top area:
- large estimated score
- band badge color
- confidence tag

Middle area:
- composite / range / gap lines
- small trend or comparison bars

Lower area:
- short action note
- nearest target callout

### Result states

1. Near-cutoff state
   - warm amber or coral emphasis
   - show buffer warning
   - say which section can move the result most

2. Safe-above-target state
   - calm green or teal emphasis
   - emphasize stability

3. Weak-score state
   - stronger warning but still student-friendly
   - point to the section with highest leverage

The result panel should visually compete with the input card, not hide behind it.

## 13. Visual chart modules

This is the biggest visual upgrade required for v3.1.
The page currently lacks enough chart-like material and color assets.

### Module A: APUSH score band chart

Add a chart or band visualization that shows approximate score bands.

Format options:

- horizontal score band ladder
- stacked band cards
- segmented progress bar
- radial/arc score indicator

The chart should communicate:

- 1 to 5 band structure
- relative cutoff feeling
- where the current estimate sits
- why a buffer matters near the boundary

### Module B: Section weighting chart

Add a chart showing APUSH section weight distribution.

Suggested representation:

- MCQ 40%
- SAQ 20%
- DBQ 25%
- LEQ 15%

Use distinct colors for each section.
This chart should sit near the calculator or just after it.

### Module C: What matters most right now

Add a small recommendation chart or card cluster that answers:

- if MCQ is weak, what should I do?
- if DBQ is weak, what should I do?
- if LEQ is weak, what should I do?
- if I’m near a cutoff, where do I gain the fastest points?

This can be a 2x2 grid of visual cards with color markers.

### Module D: Score comparison strip

Add a comparison strip for 3 / 4 / 5 targets.

Each target should show:

- band color
- estimated gap
- readiness hint

This strip is important for the “gap to target” story.

## 17. Homepage visual system

The homepage needs to feel like part of the same product family, not a separate template.

### Required hero treatment

- headline + supporting copy on one side
- featured APUSH preview / score card on the other side
- “Popular starting points” integrated below, not detached

### Popular starting points design rule

The block must stay visually connected to the hero through:

## 18. Mobile design requirements

This is a critical user complaint area.

### Mobile target widths

- 360px
- 390px
- 414px

### Mobile requirements

- no horizontal scroll
- no clipped labels
- no too-small score fields
- no crowded input rows
- no detached homepage sections
- no unreadable helper text

### Mobile calculator layout

Use a single-column stack:

1. hero
2. featured APUSH visual or preview
3. inputs
4. result card
5. charts / score bands
6. exam structure
7. study guidance
8. FAQ

Important mobile sizing rules:

- body text: 16px minimum
- input text: 16px minimum, preferably 18px
- button height: 48px minimum
- input height: 48–52px
- cards should have generous inner padding

### Mobile label handling

For labels like “Enter scores”:

- keep them short
- if they must wrap, the wrap must be intentional and visually balanced
- do not let labels wrap because the input row is too narrow
- prefer stacked labels on mobile rather than forcing a cramped horizontal row

- light surface
- tinted surface

## 21. Developer acceptance criteria

This redesign is not complete unless all of these are true:

### Visual / layout

- APUSH page width and rhythm are consistent across sections
- the APUSH page no longer looks like an inserted module
- the page includes chart-like visual assets and color-coded score bands
- the result panel feels materially richer than the previous version
- the design is visibly different from v1 and v2

### Typography / inputs

- “Enter scores” or any similar row label does not awkwardly wrap on desktop at normal widths
- mobile input text is large enough to read and edit comfortably
- mobile score fields are not cramped or tiny
- labels and helper text remain legible at 360/390/414

### Homepage integration

- homepage featured APUSH content remains attached to the hero at narrower widths
- the “Popular starting points” block never floats away visually from the hero
- homepage → hub → APUSH continuity is obvious

### Charts / visuals

- at least one score band visualization is present
- at least one section-weight visualization is present
- at least one colored recommendation module is present
- the page uses color as an information system, not just decoration

### Trust / SEO

- unofficial disclaimer remains visible
- browser-local / no-signup messaging remains present
- route links are correct
- SEO sections remain scannable
- mobile and desktop both work cleanly

## 22. Handoff summary for development

This v3.1 redesign should be implemented as a visible rebuild of the APUSH experience and homepage path.
The priority is not just fixing spacing bugs.
The priority is making the site feel more competitive, more visual, and more like a serious AP score planning product.

If anything has to be simplified, keep the calculator and score-band visuals first.
Do not simplify it back into a plain text-heavy tool page.
