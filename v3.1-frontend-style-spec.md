# AP Score Calculator v3.1 — Frontend Style Spec Based on apushscorecalculator.us

## Purpose
This document is written for front-end implementation.
It is intentionally concrete.
The developer should be able to build from this spec without guessing the visual direction.

Primary benchmark:
- https://apushscorecalculator.us/

Goal:
- Borrow the benchmark's clarity and calculator-first rhythm
- Exceed it in polish, spacing, mobile usability, and visual richness
- Avoid the prior “template-like” feel
- Make the APUSH page feel like a product page with a dashboard-like calculator experience

---

## 1. Benchmark analysis summary

### What the competitor does well
- Makes the APUSH subject feel like a dedicated product page, not a generic article.
- Keeps the calculator near the top of the page.
- Uses clear explanatory structure: calculator, scoring explanation, study guidance, FAQ.
- Supports search intent with strong on-page content.
- The page rhythm feels like a single subject experience rather than a site-wide generic matrix.

### What we must do better
- Stronger visual hierarchy and more distinctive styling.
- Clearer separation between calculator workspace and explanatory sections.
- More compelling result visualization.
- Better mobile form ergonomics.
- Richer use of color and chart-like scoring visuals.
- Stronger connection between homepage, hub, and APUSH page.
- Less generic white-card feel; more editorial product polish.

## 3. Visual direction

### Overall feel
- Clean education tool
- Calm but not plain
- Credible and modern
- Slightly editorial / slightly dashboard-like
- Not playful, not gimmicky
- More visual interest than a standard SaaS template

### Layout style
- Large container width on desktop: around 1180–1240px
- Strong section separation using vertical rhythm, not heavy dividers
- Use grids and cards, but vary density per section
- Main calculator section should feel like the page’s primary object

### Color system
Use the following palette as the implementation base:
- Page background: `#F7F4EE` or `#F8FAFC`
- Surface: `#FFFFFF`
- Surface alt: `#F3F7FB`
- Text primary: `#172033`
- Text secondary: `#667085`
- Border: `#D9E2EC`
- Primary navy: `#1D3557`
- Accent amber: `#F4A261`
- Accent teal: `#2A9D8F`
- Warning: `#E9A23B`
- Success: `#2F855A`

### Typography
- Display: `Sora` or `Space Grotesk`
- Body: `DM Sans` or `Source Sans 3`
- Numeric score styling should use a slightly heavier weight and tighter tracking

### Corner / shadow style
- Border radius: 18px for main cards, 14px for smaller cards, 999px for pills
- Shadow: soft and low elevation, not heavy drop shadows
- Borders should remain visible for structure

## 4. Homepage style spec

### Hero
The homepage hero should not be empty or overly minimal.
It needs a clear visual anchor.

Layout:
- Desktop: 2-column split
- Left: headline, subcopy, CTAs, trust line
- Right: preview card or mini dashboard visual

Hero card visual:
- Use a small score dashboard style card
- Include a sample number, score band, and small range visual
- Use a subtle accent background and a chart-like strip

The hero should feel like a real product intro, not just text on a white page.

### Featured APUSH path
The “Popular starting points” block must stay visually attached to the hero on narrow widths.

Implementation rule:
- On desktop, it can appear as a band below hero.
- On tablet/mobile, it should become part of the hero flow, not drift far away.
- Use the same container width and shared background logic.
- Avoid large vertical gap between hero and the featured chips.

### Subject matrix
Use a bento-like subject grid:
- Category heading on the left or top
- Cards with small label chip, title, and brief descriptor
- Active card / featured card can use slightly stronger background tint
- APUSH card should stand out with a more prominent accent treatment

### Visual content on homepage
Add at least one of:
- mini bar / range visualization
- small score scale strip
- visual chip group
- gradient-backed dashboard preview card

This is to avoid the page reading as pure text + buttons.

## 5. Hub style spec

Hub should feel like a clean subject browser.

### Style
- Slightly denser than homepage
- More cards, less narrative
- Category groups with strong headings
- Each card should have:
  - short subject label
  - full subject title
  - one-line descriptor
  - small scoring cue or input count

### APUSH highlight treatment
- APUSH card should have the strongest highlight within Social Science
- Use accent border or tinted background
- It should be visually obvious but not loud

## 6. APUSH page style spec

This is the core page. It must feel the most designed.

### 6.1 Hero
Requirements:
- H1 should fit on one line on wide desktop if possible
- On mobile it can wrap, but not into awkward short fragments
- Keep “Enter scores” or helper labels from wrapping awkwardly
- Use concise supporting copy
- Include a small disclaimer line, not a long paragraph

Recommended hero structure:
- title block
- one-line support copy
- small trust/meta line
- CTA row leading to calculator and explanation
- right-side dashboard card or chart card

### 6.2 Calculator workspace
This section must be highly usable.

Desktop layout:
- 2-column grid
- Left: input card(s)
- Right: result card / score dashboard
- Keep top alignment and equal heights where possible

Mobile layout:
- Stack vertically
- Inputs first, result directly after or sticky within the same flow
- Inputs must be large enough for easy tapping

### 6.3 Input styling
Inputs should be visibly larger than the current version.

Rules:
- Minimum touch target height: 48px
- Font size: 16px minimum
- Labels should stay on one line where possible
- If helper text is long, use shorter copy
- Score input groups need more width so “Enter scores” does not wrap awkwardly

Suggested input card style:
- white surface
- soft border
- title row with label + helper badge
- subtle background fill for input rows
- numeric field aligned right or center depending on layout
- clear focus ring in accent color

### 6.4 Result panel
The result panel must be more visual than the current version.

Include:
- large estimated AP score number
- band label (e.g. “Estimated 4”) 
- small composite / raw score line
- gap-to-target visualization
- confidence indicator
- colored score range strip or mini chart

Visual behavior:
- use amber for near-cutoff warning
- use teal/green for comfortable margin
- use navy / slate for neutral state

Result panel should look like a dashboard card, not a plain paragraph.

### 6.5 Score chart / band visualization
This is one of the missing competitive elements.

Add a small visualization block such as:
- horizontal range bar for 1–5 thresholds
- segmented score band strip
- callout marker showing current estimate

This should be simple enough to implement with divs/SVG, but clearly visible.

### 6.6 Score explanation block
Use an editorial card style.

Structure:
- short heading
- 2–4 concise paragraphs or bullets
- optional icon or left-side accent bar
- keep it readable and not too verbose

### 6.7 Study guidance block
This should feel actionable and not text-heavy.

Add 3 cards or 3 bullet columns:
- If MCQ is weak
- If SAQ is weak
- If DBQ/LEQ is weak

Each card should have:
- title
- one-line reason
- one-line action

### 6.8 FAQ
Use accordion cards.

Style:
- white cards on tinted section background
- rounded corners
- simple chevrons or plus icons
- strong spacing between items
- no cramped text

### 6.9 Related calculators
Use compact cards or chips.
This should visually continue the site matrix and keep internal linking strong.

## 7. Visual assets requirement

The page currently lacks enough chart/color/image feel.
This version must add at least one meaningful visual asset approach:

Option A:
- score dashboard illustration card in hero
- range chart in result panel
- small visual strip in summary section

Option B:
- abstract but product-like score visualization blocks
- no real photos needed, but the page must not look blank or purely textual

Do not use decorative images that feel fake or irrelevant.
The visual asset must support the calculator/product story.

---

## 8. Responsive rules

### Desktop
- APUSH hero and calculator should feel balanced
- Keep content width consistent across sections
- Avoid content blocks that suddenly narrow or detach from the grid
- “Enter scores” and similar labels should remain on one line if possible

### Tablet
- Reduce column gap gradually
- Maintain card alignment
- Avoid hero-to-chip detachment

### Mobile 360 / 390 / 414
- Inputs and buttons must be easy to tap
- Use 16px+ body text, 15–16px input text minimum
- Input rows should have enough vertical padding
- Result card should not shrink into a tiny unreadable block
- No horizontal overflow
- The page should remain visually interesting even when stacked

---

## 9. Concrete implementation tokens

### Spacing scale
- Section padding top/bottom: 72–96px desktop, 48–64px mobile
- Card padding: 20–24px desktop, 16–18px mobile
- Gap between hero columns: 32–40px desktop
- Gap between sections: 28–40px

### Button style
- Height: 44–48px
- Radius: 999px or 14px depending on placement
- Primary button: navy or teal fill with white text
- Secondary button: white surface with border

### Chart style
- Use simple segmentation and accent marker
- Keep it compact but visible
- Do not make the chart tiny

### Badge style
- Small rounded pills
- Soft tint backgrounds
- Text should be concise

## 10. Acceptance criteria for development

The developer should not consider this done unless:

- APUSH page width and vertical rhythm are consistent
- “Enter scores” does not awkwardly wrap into two lines on normal desktop widths
- Homepage “Popular starting points” stays visually attached to hero on narrower widths
- Mobile score input controls are bigger and easier to use
- The page clearly looks more competitive than the first version
- Charts / band visualization / score dashboard are visible
- Color use is richer than before
- The page still feels credible and educational, not gimmicky
- Desktop and mobile both feel intentional
